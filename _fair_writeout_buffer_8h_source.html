8 /**
9  * @class FairWriteoutBuffer
10  *
11  * @author Tobias Stockmanns
12  * @brief A container class to store digi data during events
13  *
14  * The FairWriteoutBuffer class provides a container which handles the storage of data between
15  * events. The data has to be given with an active time, the absolute time the data is active in the
16  * detector and can influence the output of the data.
17  * FillNewData is used to put new data into the container.
18  * WriteOutData with a given actual time stores the data which has an active time older than the given time
19  * in the corresponding TClonesArray of the FairRootManager.
20  * At the end of the task WriteOutAllData has to be called to store the data which is still in the buffer.
21  * If one adds data via FillNewData into the buffer which is already present in the buffer, Modify is called.
22  * The standard behavior of Modify is that the new data is ignored to simulate pile-up. If a different behavior
23  * is wanted one has to overwrite Modify in a derived class.
24  *
25  * The data which should be stored in the buffer has to be derived from FairTimeStamp.
26  * It needs an operator< and a method equal if the same detector element is hit.
27  *
28  * To use this buffer one has to derive his own buffer class from FairWriteoutBuffer and overwrite the pure virtual functions.
29  */
